## 職務経歴

### TRASS（業務委託）

- **期間：** 2025年11月 〜 2026年1月30日
- **形態：** 業務委託／プロダクト開発・技術設計支援

#### 概要

Web プロダクト開発支援として参画し、アーキテクチャ設計、設計思想の明文化、実装ルールの策定、既存コードベースの構造改善を中心に担当。
Next.js／React／TypeScript を用いたモダン Web 開発において、実装速度と中長期的な保守性・拡張性の両立を重視し、プロダクト全体の技術的健全性向上に貢献。

#### 詳細

##### フロントエンド設計・実装

- Next.js（App Router）を前提としたページ構成・ルーティング設計
- Server Components / Client Components の責務分離を意識した設計・実装
- 状態管理のスコープ最適化（ローカル state / props / server state の適切な切り分け）
- TypeScript を用いた型安全な設計（Props / API Response / Domain Model の型定義）
- UI コンポーネントの責務分離（Presentational / Container の分離）
- 再利用性・可読性を重視した Atomic Design ベースのコンポーネント設計

##### 設計思想・ルールの言語化（DESIGN.md 整備）

- DESIGN.md の新規策定および既存内容の再整理
- 「なぜこの設計を採用するのか」を明文化し、属人性を排除
- コンポーネント設計、ディレクトリ構成、命名規則、責務分離の基準を定義
- 実装時の判断基準（やらないこと／避ける実装）を明示し、レビューコストを削減
- Claude Code 等の AI コードレビュー前提を考慮した設計ガイドライン整備

##### コードベース改善・リファクタリング

- 肥大化したコンポーネントの責務分割と再設計
- 条件分岐・副作用が集中していたロジックの整理
- 再利用困難な実装の共通化・抽象化
- 技術的負債となっていた実装の段階的解消
- 将来的な機能追加を見据えた拡張ポイントの設計

##### UI / UX 観点での技術的改善

- UI の一貫性を担保するためのコンポーネント統一
- ユーザー操作フローを考慮した画面遷移・状態遷移設計
- アクセシビリティを意識したマークアップ・インタラクション設計
- 不要な再レンダリングの抑制、描画パフォーマンス改善

##### 開発体制・レビュー・生産性向上

- コードレビューを通じた設計観点・実装観点でのフィードバック
- 「動けば良い」ではなく「保守できるか」を軸にしたレビュー方針の共有
- 実装者間で判断がブレないための設計基準整備
- 将来的な開発者増員を見据えたオンボーディングしやすい構成への改善

短期的な機能開発に留まらず、プロダクト寿命を延ばすための技術的土台作りを主眼に置き、実装・設計・ドキュメントの三点から継続的に改善を行っている。
